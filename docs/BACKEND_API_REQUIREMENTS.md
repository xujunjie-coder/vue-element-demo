# 后端接口优化需求文档

> 项目：同花顺多端行情分析系统  
> 日期：2026-02-23  
> 背景：当前 `/spot?stock_type=ZhA` 返回全量5000+条数据，响应超时(>30s)，严重影响前端加载体验。  
> 目标：拆分为4个轻量接口，单接口响应时间控制在 **500ms 以内**。

---

## 一、现有接口问题

| 接口 | 问题 |
|---|---|
| `GET /spot?stock_type=ZhA` | 返回全部A股5000+条，响应超时>30s |
| `GET /spot?stock_type=ShA` | 返回沪市全量，同样超时 |
| `GET /stock_last?code=sz300449` | **创业板(300/301)代码返回400** |
| `GET /stock_last?code=sh688147` | **科创板(688)大部分代码返回400** |

前端多个功能共用这一个接口（表格展示、热门排行、涨跌统计、股票搜索），导致所有功能都被拖慢。

### 🔴 紧急Bug：`/stock_last` 不支持创业板和科创板

前端实测调用结果：

| 代码 | 板块 | 结果 |
|---|---|---|
| `sh600519` 贵州茅台 | 沪市主板 | ✅ 200 OK，198ms |
| `sz000001` 平安银行 | 深市主板 | ✅ 200 OK，195ms |
| `sz002594` 比亚迪 | 中小板 | ✅ 200 OK，189ms |
| `sz300449` 汉邦高科 | **创业板** | ❌ **400 错误** |
| `sz300750` 宁德时代 | **创业板** | ❌ **400 错误** |
| `sh688147` 微导纳米 | **科创板** | ❌ **400 错误** |
| `sz301269` | **创业板** | ❌ **400 错误** |

**影响范围：** 前端所有需要查询单只股票信息的功能（市盈率计算、交易模拟、自选股添加、止损计算等），只要用户输入创业板或科创板代码，就会查询失败。

**期望修复：** `/stock_last?code=sz300449` 返回与沪市主板一致的正常数据格式。

## 二、新增接口需求

### 接口1：行情分页查询

> 用途：行情首页表格按需加载，每次只取一页数据

```
GET /spot/list?stock_type=ZhA&page=1&size=50&sort=zd&order=desc
```

**请求参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| stock_type | string | 是 | - | 股票类型：`ZhA`(全部A股) / `ShA`(沪市) / `SzA`(深市) / `CyA`(创业板) |
| page | int | 否 | 1 | 页码，从1开始 |
| size | int | 否 | 50 | 每页条数，范围 10-200 |
| sort | string | 否 | code | 排序字段：`code`/`name`/`last`/`zd`/`ud`/`vol`/`turnover` |
| order | string | 否 | asc | 排序方向：`asc`(升序) / `desc`(降序) |

**响应格式：**

```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "code": "sh600519",
        "name": "贵州茅台",
        "last": 1485.30,
        "open": 1486.60,
        "close": 1486.60,
        "hod": 1507.80,
        "lod": 1470.58,
        "vol": 28956,
        "amount": 6216379203,
        "zd": -0.09,
        "ud": -1.30,
        "turnover": 3.25
      }
    ],
    "total": 5102,
    "page": 1,
    "size": 50,
    "pages": 103
  }
}
```

**关键字段说明：**
- `total`：总记录数（前端分页组件需要）
- `pages`：总页数
- `list`：当前页的数据数组，字段格式与现有 `/spot` 返回的每条数据一致

---

### 接口2：热门排行榜

> 用途：右侧栏"热门个股"模块，按涨幅/跌幅/换手率排序取Top N

```
GET /spot/top?stock_type=ShA&sort=zd&order=desc&limit=10
```

**请求参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| stock_type | string | 否 | ShA | 股票类型 |
| sort | string | 是 | - | 排序字段：`zd`(涨跌幅) / `turnover`(换手率) |
| order | string | 否 | desc | `desc`=涨幅榜/换手率榜，`asc`=跌幅榜 |
| limit | int | 否 | 10 | 返回条数，范围 1-50 |

**响应格式：**

```json
{
  "code": 200,
  "data": [
    {
      "code": "sh688147",
      "name": "微导纳米",
      "last": 94.98,
      "zd": 14.57,
      "ud": 12.08,
      "vol": 156890,
      "turnover": 35.20
    }
  ]
}
```

**前端使用场景：**
- `sort=zd&order=desc` → 涨幅榜
- `sort=zd&order=asc` → 跌幅榜
- `sort=turnover&order=desc` → 换手率榜

---

### 接口3：行情统计

> 用途：左侧栏显示"涨/跌/平"家数统计，无需返回明细

```
GET /spot/stat?stock_type=ShA
```

**请求参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| stock_type | string | 否 | ShA | 股票类型 |

**响应格式：**

```json
{
  "code": 200,
  "data": {
    "total": 2198,
    "up": 605,
    "down": 1656,
    "flat": 183,
    "limit_up": 12,
    "limit_down": 8,
    "avg_zd": -0.35
  }
}
```

**字段说明：**
- `total`：股票总数
- `up`：上涨家数（zd > 0）
- `down`：下跌家数（zd < 0）
- `flat`：平盘家数（zd = 0）
- `limit_up`：涨停家数（可选）
- `limit_down`：跌停家数（可选）
- `avg_zd`：全市场平均涨跌幅（可选）

---

### 接口4：股票搜索

> 用途：交易页面输入股票代码/名称时的模糊搜索，替代前端全量过滤

```
GET /spot/search?keyword=茅台&limit=10
```

**请求参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| keyword | string | 是 | - | 搜索关键字，支持股票代码或名称模糊匹配 |
| limit | int | 否 | 10 | 返回条数，范围 1-50 |

**响应格式：**

```json
{
  "code": 200,
  "data": [
    {
      "code": "sh600519",
      "name": "贵州茅台",
      "last": 1485.30,
      "zd": -0.09
    }
  ]
}
```

**匹配规则建议：**
- 代码前缀匹配：`6005` → 匹配 `sh600519`, `sh600508` 等
- 名称模糊匹配：`茅台` → 匹配 `贵州茅台`
- 拼音首字母匹配（可选）：`gzmt` → 匹配 `贵州茅台`

## 三、性能要求

| 指标 | 要求 |
|---|---|
| 单接口响应时间 | ≤ 500ms |
| `/spot/top` 响应时间 | ≤ 200ms（核心热门数据） |
| `/spot/stat` 响应时间 | ≤ 100ms（简单聚合） |
| `/spot/search` 响应时间 | ≤ 300ms |
| 数据更新频率 | 建议后端每3-5秒刷新一次缓存 |

**优化建议：**
1. 后端可将全量行情数据缓存在内存（Redis/进程内缓存），定时从数据源拉取更新
2. 排序和统计直接在内存中操作，避免每次请求都查数据源
3. 搜索可对股票代码和名称建索引，实现毫秒级模糊查询

| 接口 | 状态 | 说明 |
|---|---|---|

## 五、接口对接优先级

| 优先级 | 接口 | 原因 |
|---|---|---|
| 🔴 P0 | `/spot/list`（分页） | 行情首页核心功能，当前完全不可用 |
| 🔴 P0 | `/spot/top`（排行榜） | 热门个股展示，用户高频使用 |
| 🟡 P1 | `/spot/stat`（统计） | 侧边栏统计展示 |
| 🟡 P1 | `/spot/search`（搜索） | 交易页股票搜索 |

## 六、错误码约定

```json
{ "code": 200, "data": ... }          // 成功
{ "code": 400, "message": "参数错误" }  // 参数校验失败
{ "code": 500, "message": "服务异常" }  // 服务端错误
```

**对接联系：** 前端完成后会对接适配，后端接口就绪后通知前端即可。字段命名请与现有 `/spot` 返回的字段保持一致（code/name/last/zd/ud/vol/hod/lod 等），前端可无缝切换。
