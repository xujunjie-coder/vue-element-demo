# 后端回复评估与采纳（比赛版）

**评估**：可行。该策略在比赛演示场景下可采用（即：前端做“无感刷新 + 遇 401 自动续签并重试”），但生产环境需额外安全措施（如 HttpOnly cookie、refresh 旋转、CSRF 防护、refresh 滥用检测等）。

**后端已回复并将按下列要点对接（已采纳）**：
1. refresh token 的话用无感刷新（前端实现静默刷新）。
2. 401 的话自动续签并重试请求（前端需实现单飞刷新锁与请求队列，避免并发多次刷新）。
3. 后端说明：密码错误与令牌无效均返回 401；目前没有 403 场景（当前逻辑：未登陆或无权限统一返回 401）。
4. Base URL：后端会在部署到云服务器后给出具体地址，届时以该地址为准。
5. 跨域：后端建议直接前后端合并部署以避免 CORS 配置；若分离部署，后端需正确配置 CORS 与 credentials，前端可使用 axios `withCredentials` 或本地代理进行测试。
6. 前端测试：本地阶段可通过 axios 代理（dev proxy）进行测试；构建后将把静态资源交由后端托管（后端直接放入服务）以免额外配置 base url。

**Token 时效（后端确认）**：

- `access_token` 有效期：15 分钟（900 秒）。
- `refresh_token` 有效期：7 天（604800 秒）。
- `exp` 字段：以 Unix 时间戳（秒）返回，前端可据此计算静默刷新时机。
